# バックアップ・リストア手順

## 1. バックアップの構成
RDS は自動スナップショットを毎日 03:00 JST に取得し、保持期間は 7 日間。
EC2 の EBS ボリュームは AWS Backup で毎日 02:00 JST に取得し、保持期間は 30 日間。
S3 の重要バケットはバージョニングを有効化しており、誤削除は世代から復元する。

## 2. RDS のリストア
スナップショットからのリストアは新しいインスタンスとして作成される（既存への上書きではない）。
ポイントインタイムリカバリ（PITR）は直近 5 分前までの任意の時刻を指定できる。
リストア後はパラメータグループとセキュリティグループが意図どおりかを必ず確認する（デフォルトに戻ることがある）。

## 3. EBS のリストア
AWS Backup の復旧ポイントからボリュームを作成し、対象インスタンスにアタッチして差し替える。
ルートボリュームの場合は、インスタンスを停止してからデタッチ・アタッチを行う。

## 4. リストア訓練
四半期に 1 回、staging 環境でリストア訓練を実施する。
訓練では RTO 60 分・RPO 24 時間を満たせるかを計測し、結果を運用 Wiki に記録する。
