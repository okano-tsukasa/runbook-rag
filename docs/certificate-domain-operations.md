# 証明書・ドメイン運用手順

## 1. ACM 証明書の更新失敗
ACM の証明書は DNS 検証であれば自動更新される。更新失敗の通知が来たら、まず Route53 に検証用 CNAME レコードが残っているかを確認する。
メール検証の証明書は自動更新できないため、見つけたら DNS 検証への移行を起票する。
有効期限の 30 日前になっても更新されない場合は Sev2 として扱い、即日対応する。

## 2. ドメインの切り替え（Route53）
切り替え前に対象レコードの TTL を 300 秒まで下げ、浸透を待ってからレコードを変更する。
加重ルーティングを使い 10% → 50% → 100% と段階的に切り替え、各段階で 5xx エラー率とレイテンシを監視する。
切り戻しできるよう、変更前のレコード値を必ず控えてから作業する。

## 3. 期限切れを起こさないために
ACM 管理外の証明書（外部購入分）は有効期限を監視シートで管理し、60 日前にリマインダーを設定する。
証明書の失効は顧客影響が即時に出るため、期限切れを検知した場合は Sev1 として扱う。
