# EC2 トラブルシューティング手順

## SSH 接続できない
セキュリティグループの 22番ポートが、接続元IPに対して許可されているか確認する。
インスタンスのステータスチェックが 2/2 通過しているかを確認する。
EC2 コンソールの「システムログの取得」でブート失敗が出ていないか確認する。
キーペアの取り違えが多いので、対象環境の鍵を再確認する。

## ディスクフルでログインできない
別インスタンスに該当 EBS をアタッチし、マウントして不要ファイルを削除する方法が確実。
緊急時はステータスチェック後に再起動を試すが、データ整合性に注意する。

## プロセスが起動しない
systemctl status <service> で失敗理由を確認する。
journalctl -u <service> --no-pager | tail -50 で直近のログを確認する。
依存サービス（DB接続先、マウント、環境変数）の前提が満たされているかを切り分ける。

## メモリ不足（OOM）
dmesg | grep -i oom で OOM Killer の発動を確認する。
恒常的なら原因プロセスのメモリリークを疑い、インスタンスタイプ見直しを起票する。
